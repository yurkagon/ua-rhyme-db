# AGENTS.md — UA RhymeDB

Ukrainian rap rhyme database. A file-based, no-backend SPA (React + Webpack) for searching and browsing rhymes used in Ukrainian rap songs.

## Tech Stack

- **React 18** + **TypeScript** — UI
- **Webpack 5** — bundler (no Next.js, no Vite)
- **React Router 6** — client-side routing (data router, lazy routes)
- **Lodash** — data manipulation
- **Bootstrap 5** — styling base
- **CSS** — component styles

## Data Formats

### Song file — `database/lazy/songs/<id>_<authorId>_<author>_<name>`

Filename encodes four `_`-separated fields — the parser splits on `_` with no escaping, so underscores in author/title names will break it.

```
[Verse 1: Artist]
Line one of lyrics
Line two of lyrics

[Chorus]
...

###
id:glava941
author_id:glava94
author:Глава 94
name:Твоя черга
youtube:https://www.youtube.com/watch?v=z0dDvIekOrU
```

- Lines wrapped in `[...]` render as italic labels (section markers).
- Line numbers are **1-based**, counted from line 1 of the text block (before `###`).
- `youtube` is optional; the UI replaces `watch?v=` with `embed/`.
- The `###` separator is required — `loadSongByUrl` does a hard `split("###")`.

### Rhyme file — `database/bundled/rhymes/<anchor-word>`

Filename = anchor word for the group. Each line = one rhyme entry.

```
баран#r:glava942[20-21]
Степан#r:glava946[38-39]
банан
взнавай/взнавав
вставай/вставав
```

Syntax per line: `<word>[/alt1/alt2]#<type>:<info>`

- `/` — alternative forms of the same word (all alternatives resolve to this group).
- `#r:<songId>[<from>-<to>]` — mention: link to line range in a song. Multiple mentions comma-separated: `#r:glava942[20-21],glava946[38-39]`. Single-line mention: `[20]`.
- Currently only `r` info-type is implemented (`RhymeInfoType.r`).

### `database/bundled/song_list`

Auto-generated by `setup/generate_song_list.js` on every `npm run dev`/`build`. One song filename per line, sorted. **Never edit by hand.**

## Key Invariants

| Constraint | Where it breaks if violated |
|---|---|
| Song filename: exactly 4 `_`-separated fields | `StaticDatabase.loadSongList()` splits on `_` |
| `songId` in `#r:` must match first field of song filename | `StaticDatabase.getSong(songId)` returns undefined |
| Line numbers in `[from-to]` are 1-based, text block only | `MentionCard` slices with `.slice(from - 1, to)` |
| `###` must exist in every song file | `loadSongByUrl` hard-splits on `###` |
| `database/bundled/rhymes/` must contain only rhyme files | `require.context` picks up everything recursively |

## Search Logic

**`Application.findRhymes(phrase)`** — two parallel paths:

1. **`App.find(phrase)`** — scans all `rhymeData` blocks for `value === phrase` or `alternatives.includes(phrase)`. For multi-word phrases: strips the first word and recurses until single-word.
2. **`Rhymer.findRhymes(phrase)`** — phonetic fallback. Parses the input into phonetic groups (vowels/sonorous/muffled consonants), builds a regex on the last 2 chars (end-rhyme) and first 2 chars (start-rhyme), tests against all `rhymeWords` labels. Results flagged as `algorithmic: true`.

Results are merged with `_.uniqBy("label")` — curated entries take precedence over algorithmic.

**`formatWord(word)`** normalises before lookup: lowercase, strips `'`, `'`, `"`, `` ` ``, `»`, `«`, replaces `-` with space.

## StaticDatabase Lifecycle

`StaticDatabase.load()` is called at **module import time** (bottom of `StaticDatabase.ts`) — synchronous, blocks until all rhyme data is parsed. Three caches:
- `rhymeData: RhymeBlock[]` — grouped rhyme blocks as parsed from files.
- `rhymeWords: Rhyme[]` — flat deduplicated list of all words (for `Rhymer`).
- `songList: Song[]` — song metadata + lazy asset URLs.

Song lyrics (`song.data`) are loaded lazily via `StaticDatabase.loadFullSongData(songId)` — fetch on first access, cached in-place on the `Song` object.

## Build Notes

- `webpack.common.js` calls `require("./generate_song_list")` synchronously — the song index is regenerated on every dev/build start.
- `database/bundled/*` → `type: "asset/source"` — file contents inlined as strings.
- `database/lazy/*` → `type: "asset/resource"` — files copied to output, `require()` returns public URL.
- Production `publicPath` is derived from `package.json#homepage` for GitHub Pages routing.
- Two `HtmlWebpackPlugin` instances emit both `index.html` and `404.html` (GitHub Pages SPA routing workaround).

## Running Locally

```bash
npm install
npm run dev      # http://localhost:3000
npm run build    # production build → build/
npm run serve    # serve build/ on http://localhost:3001
```

## What to Avoid

- Do not add files with extensions to `database/bundled/rhymes/` — `require.context` picks up everything.
- Do not rename `song_list` — `RawFileExtractor.loadSongList()` hardcodes the path.
- Do not make `StaticDatabase.load()` async without also making `Application.findRhymes()` async and updating the `Search` page (`useMemo` → `useEffect`).
- Do not use `_` in author names or song titles — the filename parser has no escape mechanism.
